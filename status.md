## Component‑by‑Component Explanation

### `Execution/__init__.py`
```python

```
*Purpose*: Enables imports such as `from Execution.newsextractor import …`.

### `Execution/newsextractor.py`
**Purpose**: Runs the end‑to‑end news‑alert extraction pipeline.
1. Initialise the SQLite DB (`processed_files.db`) to avoid duplicate work.
2. Iterate over each JSON file in `parsed_output/`.
3. Build a prompt that embeds the **Pydantic** schema from `schemas.py`.
4. Call the LLM via OpenRouter.
5. Validate the JSON response against `FilingNewsExtraction`.
6. Write the validated result to `news_output/<symbol>/…_news.json`.
7. Log token usage with `token_tracker.record_token_usage`.
8. At the end, print a spend summary.

### `Execution/schemas.py`
```python
from pydantic import BaseModel, Field
from typing import List

class Fact(BaseModel):
    alert_message: str = Field(..., description="One‑sentence news‑style alert")
    event_category: str = Field(..., description="High‑level category, e.g., earnings, M&A")
    page_number: int = Field(..., description="Source page number in the filing")
    verbatim_source_quote: str = Field(..., description="Exact quote from the filing for traceability")
    reporting_period: str | None = Field(None, description="Fiscal period the fact refers to")

class FilingNewsExtraction(BaseModel):
    facts: List[Fact]
    has_material_development: bool = Field(..., description="True if any fact is market‑moving")
```
*How it works*: The schema is inserted into the LLM prompt (`newsextractor.build_prompt`). After the LLM replies, the response is validated against this model, guaranteeing a consistent output format.

### `Execution/token_tracker.py`
**Purpose**: Stores LLM token usage and approximated cost in `token_usage.db`. It also provides a printable summary.
Key functions:
```python
def record_token_usage(layer, model, symbol, context_label, usage, success, log=None):
    # Calculates cost from the token counts and writes a row to the DB.
    ...
```
```python
def print_usage_summary(db_path: str = USAGE_DB):
    """Prints a table of calls, token counts and cost per model."""
    if not Path(db_path).exists():
        return
    conn = sqlite3.connect(db_path)
    rows = conn.execute(
        """
        SELECT model, COUNT(*) AS calls,
               SUM(prompt_tokens) AS in_tokens,
               SUM(completion_tokens) AS out_tokens,
               SUM(cost_usd) AS cost
        FROM token_usage
        GROUP BY model
        ORDER BY cost DESC
        """
    ).fetchall()
    conn.close()
    if not rows:
        return
    print("\n--- Token usage summary ---")
    total = 0.0
    for model, calls, in_tok, out_tok, cost in rows:
        cost = cost or 0.0
        total += cost
        print(f"{model:30s} calls={calls:<5d} in={in_tok:<8d} out={out_tok:<8d} cost=${cost:.4f}")
    print(f"{'TOTAL':30s} cost=${total:.4f}")
    print("----------------------------\n")
```

### `Dataextraction/Parsing.py`
**Purpose**: Turns raw PDFs / HTML filings into a structured JSON format.
Typical steps:
1. Load the source file.
2. Extract text (using `pdfminer`, `BeautifulSoup`, etc.).
3. Chunk the document into logical sections.
4. Emit a JSON object like:
```json
{
  "filename": "ABC_20230619.pdf",
  "symbol": "ABC",
  "chunks": [{"page_num": 1, "type": "text", "content": "..."}, ...]
}
```
The resulting files live in `parsed_output/` and are consumed by `newsextractor.py`.

### `Datapulling/announcements_nse.py`
**Purpose**: Retrieves the latest NSE announcements via public API or web‑scraping and stores them as raw PDFs/JSON together with a `.meta.json` side‑car.
```python
import requests, os
API_URL = "https://www.nseindia.com/api/..."
resp = requests.get(API_URL, headers={"User-Agent": "…"})
for item in resp.json()["announcements"]:
    pdf_url = item["pdfLink"]
    pdf_bytes = requests.get(pdf_url).content
    out_path = Path("downloaded_files") / item["symbol"] / Path(pdf_url).name
    out_path.parent.mkdir(parents=True, exist_ok=True)
    out_path.write_bytes(pdf_bytes)
```

## How the Pieces Fit Together
1. **Data collection** – `announcements_nse.py` downloads raw filings into `downloaded_files/`.
2. **Parsing** – `Parsing.py` reads those files, extracts text, and writes structured JSON to `parsed_output/`.
3. **Extraction** – `newsextractor.py` walks `parsed_output/`, builds prompts, calls the LLM, validates the response, writes the final news JSON to `news_output/`, and records the file as processed in `processed_files.db`.
4. **Telemetry** – Every LLM call is logged via `token_tracker.py`. After a batch run, `print_usage_summary()` prints a spend report.
5. **Environment** – `.env` supplies the OpenRouter API key; `load_dotenv()` makes it available to the extractor.
