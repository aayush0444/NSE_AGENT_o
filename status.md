### `Execution/newsextractor.py`
**Purpose**: Orchestrates the end‑to‑end news‑alert extraction pipeline.
1. **Initialises SQLite DB** (`processed_files.db`) to avoid duplicate processing.
2. **Iterates** over JSON files in `parsed_output/`.
3. **Builds a prompt** for the LLM using the parsed filing and a JSON schema from `schemas.py`.
4. **Calls the LLM** via OpenRouter (`httpx` POST).
5. **Parses the LLM response** back into a `FilingNewsExtraction` model.
6. **Writes the result** to `news_output/<symbol>/…_news.json`.
7. **Logs token usage** with `token_tracker.record_token_usage`.
8. **Summarises** token spend at the end.

```python
def call_llm(prompt: str, symbol: str, context_label: str) -> str:
    try:
        r = httpx.post(
            "https://openrouter.ai/api/v1/chat/completions",
            headers={"Authorization": f"Bearer {OPENROUTER_KEY}", "Content-Type": "application/json"},
            json={"model": MODEL, "messages": [{"role": "user", "content": prompt}], "temperature": 0.0},
            timeout=120,
        )
        r.raise_for_status()
        body = r.json()
        usage = body.get("usage")
        content = body["choices"][0]["message"]["content"]
        success = True
    except Exception as e:
        log.error(f"LLM call failed: {e}")
        success = False
        usage = None
        content = ""
    finally:
        record_token_usage(
            layer="news_extraction",
            model=MODEL,
            symbol=symbol,
            context_label=context_label,
            usage=usage,
            success=success,
            log=log,
        )
    return content
```

### `Execution/schemas.py`
**Purpose**: Defines the JSON schema that the LLM must return. Uses **Pydantic v2** for validation.
```python
from pydantic import BaseModel, Field
from typing import List

class Fact(BaseModel):
    alert_message: str = Field(..., description="One‑sentence news‑style alert")
    event_category: str = Field(..., description="High‑level category (e.g., earnings, M&A)")
    page_number: int = Field(..., description="Source page number in the filing")
    verbatim_source_quote: str = Field(..., description="Exact quote from the filing for traceability")
    reporting_period: str | None = Field(None, description="Fiscal period the fact refers to")

class FilingNewsExtraction(BaseModel):
    facts: List[Fact]
    has_material_development: bool = Field(..., description="True if any fact is market‑moving")
```
*How it works*: `newsextractor.build_prompt()` inserts `FilingNewsExtraction.model_json_schema()` into the system prompt. After the LLM replies, `parse_llm_response()` validates the JSON against this model, guaranteeing a consistent output format.

### `Execution/token_tracker.py`
**Purpose**: Records LLM token usage and approximated cost to a SQLite DB (`token_usage.db`). Provides a summary printer.
Key functions:

```python
def init_usage_db():
    conn = sqlite3.connect(USAGE_DB)
    conn.execute("""
        CREATE TABLE IF NOT EXISTS token_usage (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            timestamp TEXT NOT NULL,
            layer TEXT NOT NULL,
            model TEXT NOT NULL,
            symbol TEXT,
            context_label TEXT,
            prompt_tokens INTEGER,
            completion_tokens INTEGER,

            cost_usd REAL,
            success INTEGER

```
```python
def record_token_usage(layer, model, symbol, context_label, usage, success, log=None):
    init_usage_db()
    usage = usage or {}
    prompt_tokens = usage.get("prompt_tokens", 0) or 0
    completion_tokens = usage.get("completion_tokens", 0) or 0
    total_tokens = usage.get("total_tokens", prompt_tokens + completion_tokens) or 0
    price = PRICE_PER_1M_TOKENS.get(model, DEFAULT_PRICE)
    cost_usd = (prompt_tokens / 1_000_000) * price["input"] + (completion_tokens / 1_000_000) * price["output"]
    conn = sqlite3.connect(USAGE_DB)
    conn.execute(
        """
        INSERT INTO token_usage
        (timestamp, layer, model, symbol, context_label,
         prompt_tokens, completion_tokens, total_tokens, cost_usd, success)
        VALUES (?,?,?,?,?,?,?,?,?,?)
        """,
        (datetime.now().isoformat(), layer, model, symbol, context_label,
         prompt_tokens, completion_tokens, total_tokens, cost_usd,
         1 if success else 0),
    )
    conn.commit()
    conn.close()
    msg = f"[tokens] {context_label} | model={model} | in={prompt_tokens} out={completion_tokens} total={total_tokens} | cost=${cost_usd:.5f} | success={success}"
    if log:
        log.info(msg)
    else:
        print(msg)
```
```python
def print_usage_summary(db_path: str = USAGE_DB):
    if not Path(db_path).exists():
        return
    conn = sqlite3.connect(db_path)
    rows = conn.execute(
        """
        SELECT model, COUNT(*) AS calls,
               SUM(prompt_tokens) AS in_tokens,
               SUM(completion_tokens) AS out_tokens,
               SUM(cost_usd) AS cost
        FROM token_usage
        GROUP BY model
        ORDER BY cost DESC
        """
    ).fetchall()
    conn.close()
    if not rows:
        return
    print("\n--- Token usage summary ---")
    grand_total = 0.0
    for model, calls, in_tok, out_tok, cost in rows:
        cost = cost or 0.0
        grand_total += cost
        print(f"{model:30s} calls={calls:<5d} in={in_tok:<8d} out={out_tok:<8d} cost=${cost:.4f}")
    print(f"{'TOTAL':30s} cost=${grand_total:.4f}")
    print("----------------------------\n")
```

### `Dataextraction/Parsing.py`
**Purpose**: Converts raw filing PDFs / HTML into a structured JSON format consumed by the news extractor. Typical steps:
1. **Load** the source file.
2. **Extract text** (using `pdfminer`, `BeautifulSoup`, or similar).
3. **Chunk** the document into pages/sections.
4. **Emit** a JSON object:
```json
{
  "filename": "ABC_20230619.pdf",
  "symbol": "ABC",
  "chunks": [{"page_num": 1, "type": "text", "content": "..."}, ...]
}
```
The resulting files live in `parsed_output/` and are the direct input for `newsextractor.py`.

## How the pieces fit together
1. **Data collection** – `announcements_nse.py` downloads raw filings into `downloaded_files/` and writes side‑car metadata.
2. **Parsing** – `Parsing.py` reads those files, extracts text, and writes structured JSON to `parsed_output/`.
3. **Extraction** – `newsextractor.py` walks `parsed_output/`, builds prompts, calls the LLM, validates with `schemas.py`, writes news JSON to `news_output/`, and marks the file as processed in `processed_files.db`.
4. **Telemetry** – Every LLM call is logged via `token_tracker.py`, which stores token counts & cost in `token_usage.db`. After a batch run, `print_usage_summary()` prints a spend report.
5. **Environment** – `.env` supplies the OpenRouter key; `load_dotenv()` makes it available to the extractor.
